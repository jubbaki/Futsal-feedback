# 즐거운 풋살을 위한 소통창구 제작 프롬프트

풋살 수업 피드백을 오래 운영할 수 있는 웹페이지를 만들어줘.

스택은 알아서 적절하게 선택해줘. 단, 오래 운영하기 좋고, 나중에 배포와 데이터 관리가 쉬운 구조였으면 좋겠어.

## 서비스 목적

나는 풋살을 하고 있고, 총 2개의 팀에 소속되어 있어.

- 함무라비
- 버브

팀원들이 코치님에게 바라는 점, 좋았던 점, 수업 만족도 등을 부담 없이 남길 수 있는 소통창구를 만들고 싶어.

익명성을 기본으로 보장해서 팀원들이 솔직하게 의견을 남길 수 있게 하고, 나는 관리자로서 전체 의견을 확인한 뒤 코치님에게 보여줄 의견과 공개 범위를 조절하고 싶어.

## 페이지 이름

`즐거운 풋살을 위한 소통창구`

## 필요한 페이지 구조

### 1. 피드백 제출 페이지

팀원들이 접속해서 의견을 제출하는 페이지.

필수 입력 항목:

- 팀 선택
  - `함무라비`
  - `버브`
- 수업 만족도
  - `0~5` 중 선택
- 수업에서 바라는 점
  - 자유롭게 글로 직접 입력

선택 입력 항목:

- 이름 또는 닉네임
  - 기본은 익명
  - 이름을 남기고 싶은 사람만 선택적으로 입력

반드시 들어가야 하는 안내 문구:

> 기본적으로 익명으로 전달돼요. 이름을 남기고 싶은 경우에만 선택적으로 적어주세요.

자유 의견 입력 문구는 아래 톤으로 작성:

> 더 즐겁고 좋은 수업을 위해 바라는 점이 있다면 편하게 적어주세요.

제출 완료 후 문구:

> 의견이 잘 전달됐어요. 더 즐거운 풋살을 만드는 데 참고할게요.

한 사람이 여러 번 제출해도 괜찮아. 매주 수업이 있기 때문에 수업마다 피드백을 남길 수 있어야 해.

### 2. 관리자 페이지

내가 보는 관리자 페이지.

관리자 페이지는 비밀번호 입력 후 들어가게 해줘.

- 관리자 비밀번호: `0413`

관리자 페이지에서 가능한 기능:

- 전체 제출 데이터 확인
- 팀별 필터
  - 전체
  - 함무라비
  - 버브
- 날짜/제출 시각 확인
- 만족도 확인
- 만족도 평균 확인
- 개별 의견 목록 확인
- 이름 또는 닉네임이 입력된 경우 확인
- 각 의견별로 코치님에게 공개할지 여부 설정
- 각 의견별로 만족도를 코치님에게 공개할지 여부 설정
- 각 의견별로 이름/닉네임을 코치님에게 공개할지 여부 설정

관리자는 원본 데이터를 모두 볼 수 있어야 해.

중요한 점:

- 코치님에게 모든 의견이 자동 공개되면 안 돼.
- 관리자가 `코치님 공개`로 설정한 의견만 코치님 페이지에 보여야 해.
- 만족도와 이름/닉네임도 관리자가 공개 여부를 따로 선택할 수 있어야 해.
- 팀명은 코치님에게 보여줘도 돼.

### 3. 코치님 페이지

코치님에게 공유할 별도 페이지.

코치님 페이지는 별도 로그인은 없어도 돼. 링크를 코치님에게만 공유할 예정이야.

코치님 페이지에서 가능한 기능:

- 관리자가 공개 처리한 의견만 보기
- 팀 필터 선택
  - 전체
  - 함무라비
  - 버브
- 팀명 표시
- 의견 내용 표시
- 관리자가 허용한 경우에만 만족도 표시
- 관리자가 허용한 경우에만 이름/닉네임 표시

초기에는 코치님 화면에 통계는 없어도 돼.

코치님은 일단 개별 의견 목록만 보면 돼.

## 권한과 공개 범위

권한은 아래처럼 나누고 싶어.

### 일반 사용자

- 피드백 제출만 가능
- 기존 제출 목록은 볼 수 없음
- 관리자 화면 접근 불가

### 관리자

- 비밀번호 `0413` 입력 후 접근
- 모든 원본 데이터 확인 가능
- 코치님에게 보여줄 의견 선택 가능
- 만족도 공개 여부 선택 가능
- 이름/닉네임 공개 여부 선택 가능

### 코치님

- 별도 로그인 없이 공유 링크로 접근
- 관리자가 공개한 의견만 볼 수 있음
- 전체/팀별 필터 가능
- 관리자가 공개하지 않은 만족도나 이름은 볼 수 없음

## 데이터 구조

피드백 데이터에는 최소한 아래 필드가 필요해.

- `id`
- `created_at`
- `team`
  - 함무라비 또는 버브
- `rating`
  - 0~5
- `name`
  - 선택 입력
  - 없으면 null 또는 빈 값
- `message`
  - 수업에서 바라는 점
- `visible_to_coach`
  - 코치님 공개 여부
- `show_rating_to_coach`
  - 만족도 코치님 공개 여부
- `show_name_to_coach`
  - 이름/닉네임 코치님 공개 여부

## 디자인 방향

풋살 느낌은 살리되 너무 장난스럽지 않았으면 좋겠어.

운영용 페이지이기 때문에 깔끔하고 편하게 써야 해.

특히 모바일에서 사용하기 좋아야 해. 팀원들이 휴대폰으로 링크를 눌러 바로 제출할 가능성이 높아.

디자인 톤:

- 따뜻하고 부담 없는 분위기
- 깔끔한 폼
- 버튼은 누르기 쉽게
- 팀 선택과 만족도 선택은 직관적으로
- 관리자 화면은 데이터를 빠르게 훑어볼 수 있게
- 코치님 화면은 개별 의견을 읽기 편하게

## UX 세부사항

피드백 제출 페이지:

- 팀 선택은 버튼 형태가 좋음
- 만족도 0~5도 버튼 형태가 좋음
- 제출 버튼은 명확하게
- 제출 후 성공 메시지를 보여줘야 함
- 입력이 부족하면 친절한 오류 메시지를 보여줘야 함

관리자 페이지:

- 처음에는 비밀번호 입력 화면
- 비밀번호가 맞으면 데이터 목록 표시
- 의견 카드 또는 테이블 형태로 보기 좋게 표시
- 각 의견마다 공개 설정 토글 또는 체크박스 제공
- 팀별 필터 제공
- 만족도 평균과 의견 수 정도는 상단에서 볼 수 있으면 좋음

코치님 페이지:

- 전체/함무라비/버브 필터 제공
- 공개된 의견이 없으면 빈 상태 문구 표시
- 관리자가 공개한 항목만 표시

## 운영상 중요한 점

이 서비스는 한 번 쓰고 끝나는 설문이 아니라, 매주 수업 이후 계속 사용할 수 있는 피드백 창구야.

그래서 아래가 중요해.

- 데이터가 계속 쌓여야 함
- 관리자가 원본을 볼 수 있어야 함
- 코치님에게 보여줄 내용은 관리자가 조절할 수 있어야 함
- 팀별로 의견을 볼 수 있어야 함
- 익명성이 기본으로 보장되어야 함
- 이름을 쓰고 싶은 사람만 선택적으로 이름을 남길 수 있어야 함
